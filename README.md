# Software Engineering Labs

This repository contains the work developed during the **Software Engineering** course. It includes the laboratory assignments completed throughout the semester, the **Software Requirements Specification (SRS)** of the **Gym Management System**, and the project folder containing the updated diagrams created for that system as the course progressed.

The repository is organized to reflect the main analysis and design artifacts produced during the different laboratory sessions.

---

## Repository Structure

### 1. Lab2_UseCase_Diagrams
This folder contains the material developed for **Lab 2**, focused on **UML Use Case Diagrams**.

These diagrams model the functional interactions between the system and its actors, identifying the main services provided by the Gym Management System and the relationships between users and system functionalities.

Main concepts covered in this lab:
- Actors
- Use cases
- System boundary
- <<include>>
- <<extend>>

---

### 2. Lab3_Class_Diagrams
This folder contains the work produced for **Lab 3**, focused on **UML Class Diagrams**.

These diagrams describe the static structure of the system and the relationships between the main domain entities involved in the Gym Management System.

Main concepts covered in this lab:
- Classes and attributes
- Associations
- Multiplicity
- Generalization
- Aggregation and composition

---

### 3. Lab4_Behavioral_Diagrams
This folder contains the deliverables for **Lab 4**, focused on **behavioral UML diagrams**.

This lab includes:
- Sequence Diagrams
- State Diagrams
- Activity Diagrams

These diagrams were used to model interaction flows, state transitions, and process behavior in different systems, and also contributed to the behavioral modeling of the Gym Management System.

---

### 4. SRS
The `SRS` folder contains the **Software Requirements Specification** for the **Gym Management System**, based on the IEEE 830 template.

This document includes:
- Introduction and purpose
- Overall description of the system
- Product features
- User classes and characteristics
- External interface requirements
- Functional requirements
- Nonfunctional requirements
- Revision history showing the progressive evolution of the specification

The SRS has been updated incrementally as new concepts and deliverables were introduced in the course laboratories.

---

### 5. project
The `project` folder contains the project artifacts related to the **Gym Management System**.

This folder includes the examples and updated versions of the diagrams created for the system throughout the semester. It contains the project modeling work refined week by week as new laboratory tasks were completed.

Typical contents of this folder include:
- Use case diagrams
- Class diagrams
- Behavioral diagrams
- Updated versions of system models

This folder represents the practical evolution of the Gym Management System project.

---

## Main Project: Gym Management System

The central project documented in this repository is the **Gym Management System**, a software solution intended to support the daily management of a gym.

The system is designed to include functionalities such as:
- Member registration and profile management
- Trainer management
- Class scheduling
- Class booking
- Attendance monitoring
- Membership validation
- Payment control

Throughout the course, the project has been progressively refined by applying software engineering techniques for requirements analysis, UML modeling, and structured documentation.

---

## Purpose of the Repository

This repository serves as a structured record of the work developed in the course. It brings together both the laboratory assignments and the documentation of the Gym Management System project, showing how the system analysis and design evolved over time.
