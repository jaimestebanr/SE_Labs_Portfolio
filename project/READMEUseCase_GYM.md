This use case diagram represents a Gym Management System and the interactions between its main actors.

The system includes three actors: Member, Administrator, and Trainer. The Member can book classes, view available classes, update their profile, and cancel bookings. The Administrator is responsible for registering members, managing payments, and assigning trainers. The Trainer can view assigned classes and track attendance.

Some operations include membership validation to ensure that only active members can access system services. Overall, the diagram provides a clear overview of the system’s functionalities and how users interact with it.
